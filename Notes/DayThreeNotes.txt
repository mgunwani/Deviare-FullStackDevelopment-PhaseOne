HTML5 Fundamentals

	- HTML is a HyperText Markup Language.
	- Helps us to create web pages for any websites.
	
<!DOCTYPE html>

	- Document Type Declaration.
	- Instructs the web browser that this document is an HTML document.
	- It is case insensitive.

-------------------------------

Types of Markup Tags:

	1. Container Tags
	
		Any markup tag that has start and end tag is container tag.
	
		<h1>....</h1>
		<p>.....</p>
		<table>....</table>
	
	2. Empty Tags (Self Closing Tags)
	
		Any markup that are self closing or not mandatory to end them are empty tags.
		
		<hr> or <hr />
	
-------------------------------
	
HTML5 Heading Tags:

	Heading Tags are container tags.
	Container Tags that needs to close.
	<h1> to <h6>
	
	<h1>Heading One</h1>
    <h2>Heading Two</h2>
    <h3>Heading Three</h3>
    <h4>Heading Four</h4>
    <h5>Heading Five</h5>
    <h6>Heading Six</h6>

HTML Lists:

	1. Unordered Lists: With no specific order. e.g. bullets.
		Types: disc, circle and square
	
		 <ul type="square">
			<li>Input Devices
				<ul>
					<li>Keyboard</li>
					<li>Mouse</li>
					<li>Scanner</li>
					<li>Microphone</li>
				</ul>
			<li>Output Devices
			<li>Storage Devices
		</ul>
	
	2. Ordered Lists: With Specific order. e.g. numbers, alphabets etc.
			Types: 1, A, a, I, i also start=3 (start attribute should use number)
	
		<ol type="i">
			<li>Input Devices</li>
			<ul>
				<li>Keyboard</li>
				<li>Mouse</li>
				<li>Scanner</li>
				<li>Microphone</li>
			</ul>
			<li>Output Devices</li>
			<li>Storage Devices</li>
			<li>Input Devices</li>
			<li>Output Devices</li>
			<li>Storage Devices</li>
			<li>Input Devices</li>
			<li>Output Devices</li>
			<li>Storage Devices</li>
		</ol>
	
	3. Description Lists: To list of terms and their descriptions.
	
		<dl>
			<dt>Input Devices</dt>
			<dd>These devices help me to take input from end user.</dd>
			<dt>Input Devices</dt>
			<dd>These devices help me to give output to the end user.</dd>
			<dt>Storage Devices</dt>
			<dd>These devices help me to store the data.</dd>
		</dl>
